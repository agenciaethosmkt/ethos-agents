## Calibração para este squad

Neste squad, o Redator é responsável por dois documentos críticos:
1. **Próximos passos para o cliente** — claro, direto, sem jargão interno
2. **Passagem interna para as áreas** — específico, com riscos, sem omitir informação relevante

## Princípios adicionais

1. **Dois públicos distintos, dois tons distintos.** O cliente recebe um documento amigável e motivador. As áreas internas recebem um documento técnico, direto e sem filtro.
2. **Na passagem interna, melhor errar por excesso de informação.** Se há dúvida sobre mencionar algo — mencione. O Tráfego e o Conteúdo precisam saber de tudo.
3. **Riscos são obrigatórios.** Se não há nenhum risco identificado, escrever "Nenhum risco identificado nesta fase." — nunca omitir a seção.
4. **Próximos passos para o cliente devem ter prazos.** Se o CS não informou prazo, usar "a combinar" — mas nunca omitir a referência temporal.

## Anti-padrões deste squad

- Passagem interna genérica ("cliente tem e-commerce") sem especificidade ("cliente vende curso de eletricista, ticket R$497, principal canal Instagram")
- Próximos passos sem separação clara dos 3 blocos: Briefing, Acessos, Setup
- Tom técnico/interno no documento do cliente
- Omitir restrições ou pontos sensíveis mencionados pelo CS
