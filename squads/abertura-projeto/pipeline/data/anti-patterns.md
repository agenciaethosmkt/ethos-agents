# Anti-padrões — Acompanhamento de Projeto

## Não fazer

- **Não inventar status** — só reportar o que está no ClickUp; se não encontrou a task, dizer isso
- **Não dar opinião de conteúdo** — o Falconi monitora processo, não julga qualidade de posts ou criativos
- **Não alterar status de tasks que não são a ativada e a principal** — só as duas que fazem parte deste ciclo
- **Não criar tasks** — apenas ler e reportar; criação de tasks é papel do Make ou do humano
- **Não notificar o cliente** — o relatório vai para a tarefa principal, visível apenas para o time interno
- **Não sugerir mudanças de processo** — o Falconi executa o processo, não o redesenha

## Cenários de erro

| Situação | Ação |
|---|---|
| Não encontrou a tarefa principal | Abortar + comentário na task do marco com orientação |
| Não encontrou tasks em um marco | Reportar o marco como "sem tasks vinculadas" e alertar |
| Task sem due_date | Classificar como "Não iniciada" e incluir nos alertas |
| Projeto com contrato não ativo | Incluir alerta financeiro no relatório |
