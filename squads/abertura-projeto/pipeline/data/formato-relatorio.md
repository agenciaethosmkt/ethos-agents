# Formato de Relatório — Acompanhamento de Projeto

## Princípios

- Relatório é para gestor humano revisar — direto, sem enrolação
- Sempre mostrar o que está OK e o que não está
- Próximos passos devem ser acionáveis (quem faz o quê)
- Máximo 1 tela de leitura — não precisa ser exaustivo

## Emojis de status

| Emoji | Significado |
|---|---|
| ✅ | Concluída / finalizada |
| 🔄 | Em andamento / execução |
| ⏳ | Não iniciada / backlog |
| 🔴 | Atrasada (due_date vencido) |
| 👁️ | Aguardando revisão |

## Nível de detalhe por marco ativado

**Marco 1 (Kickoff):** foco em verificar se toda a base está criada antes de avançar para Ativação.

**Marco 2 (Ativação):** foco em verificar se a produção está fluindo e se o Marco 1 foi 100% concluído.

**Marco 3 (Monitoramento):** foco em resultados e continuidade — relatórios sendo entregues, otimizações happening.

## Tom

- Gestão → objetivo, baseado em fatos, sem julgamento
- Alertas → diretos, sem suavizar
- Próximos passos → sempre com verbo no imperativo: "Concluir", "Verificar", "Cobrar", "Aprovar"
